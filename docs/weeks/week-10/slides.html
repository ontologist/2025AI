What is Supervised Learning?

教師あり学習とは？

Definition / 定義:

Learning from labeled examples to make predictions on new, unseen data

ラベル付き例から学習し、新しい未見データで予測を行う

Training Process / 訓練プロセス: Input (Features) + Output (Labels) → Model → Predictions 入力（特徴） + 出力（ラベル） → モデル → 予測 Example: Email text + Label (Spam/Not Spam) → Spam Filter → Classify new emails

Classification vs Regression

分類 vs 回帰

Classification / 分類

Predicting discrete categories/classes

離散カテゴリ/クラスの予測

Examples / 例:

  • Email: Spam or Not Spam
  • Image: Cat, Dog, or Bird
  • Medical: Disease Present or Absent
  • Credit: Approved or Denied

Output: Class label (discrete)

出力: クラスラベル（離散的）

Regression / 回帰

Predicting continuous numerical values

連続的な数値の予測

Examples / 例:

  • House price: $250,000
  • Temperature: 23.5°C
  • Stock price: $145.32
  • Sales forecast: 1,250 units

Output: Numeric value (continuous)

出力: 数値（連続的）

The Supervised Learning Workflow

教師あり学習ワークフロー

1. Collect Data → 2. Prepare Data → 3. Split Data → 4. Train Model → 5. Evaluate → 6. Deploy データ収集 データ準備 データ分割 モデル訓練 評価 デプロイ

Step-by-Step:

  1. Collect: Gather labeled data (inputs + correct outputs)
  2. Prepare: Clean, transform, and feature engineer
  3. Split: Divide into training, validation, test sets
  4. Train: Model learns patterns from training data
  5. Evaluate: Test performance on unseen data
  6. Deploy: Use model in production

ステップバイステップ:

  1. 収集: ラベル付きデータを集める（入力 + 正しい出力）
  2. 準備: クリーニング、変換、特徴エンジニアリング
  3. 分割: 訓練、検証、テストセットに分割
  4. 訓練: モデルが訓練データからパターンを学習
  5. 評価: 未見データでパフォーマンスをテスト
  6. デプロイ: 本番環境でモデルを使用

Understanding Training Data

訓練データの理解

Data Split Ratios / データ分割比率:

Training Set: 70-80% - Learn patterns / パターンを学習 Validation Set: 10-15% - Tune parameters / パラメータを調整 Test Set: 10-15% - Final evaluation / 最終評価

Why Split Data?

  • Training: Model learns from this data
  • Validation: Tune hyperparameters, prevent overfitting
  • Test: Unbiased evaluation of final model

Key Rule: NEVER use test data during training!

なぜデータを分割するか？

  • 訓練: モデルがこのデータから学習
  • 検証: ハイパーパラメータを調整、過学習を防ぐ
  • テスト: 最終モデルの偏りのない評価

重要なルール: 訓練中にテストデータを使用しないこと！

Features and Labels

特徴とラベル

Features (X) / 特徴

Input variables used for prediction

予測に使用される入力変数

Examples:

  • Email spam: word frequency, sender, links
  • House price: size, location, bedrooms
  • Medical: age, symptoms, test results

Good features are:

  • Relevant to the task
  • Easy to obtain
  • Informative

Labels (y) / ラベル

Output/target we want to predict

予測したい出力/ターゲット

Examples:

  • Email spam: "Spam" or "Not Spam"
  • House price: $450,000
  • Medical: "Disease" or "Healthy"

Label types:

  • Classification: Categories
  • Regression: Numbers

Overfitting vs Underfitting

過学習 vs 学習不足

Overfitting / 過学習

Model memorizes training data but fails on new data

モデルが訓練データを記憶するが新データで失敗

  • Too complex model
  • 過度に複雑なモデル
  • High accuracy on training
  • 訓練では高精度
  • Low accuracy on test
  • テストでは低精度

Solution: More data, regularization, simpler model

Underfitting / 学習不足

Model too simple to capture patterns

モデルが単純すぎてパターンを捉えられない

  • Too simple model
  • 単純すぎるモデル
  • Low accuracy on training
  • 訓練でも低精度
  • Low accuracy on test
  • テストでも低精度

Solution: More complex model, better features

Model Evaluation Metrics

モデル評価指標

Classification Metrics

分類指標

  • Accuracy: % of correct predictions
    正確度: 正しい予測の割合
  • Precision: True Positives / Predicted Positives
    精度: 真陽性 / 予測陽性
  • Recall: True Positives / Actual Positives
    再現率: 真陽性 / 実際の陽性
  • F1-Score: Harmonic mean of precision & recall
    F1スコア: 精度と再現率の調和平均

Regression Metrics

回帰指標

  • MAE: Mean Absolute Error
    MAE: 平均絶対誤差
  • MSE: Mean Squared Error
    MSE: 平均二乗誤差
  • RMSE: Root Mean Squared Error
    RMSE: 二乗平均平方根誤差
  • R²: Coefficient of determination
    R²: 決定係数

Practical Tips for Supervised Learning

教師あり学習の実践的なヒント

Best Practices:

  • More Data = Better: Generally improves performance
  • Quality > Quantity: Clean, accurate labels are critical
  • Feature Engineering: Create informative features
  • Start Simple: Try simple models first (baseline)
  • Cross-Validation: Use k-fold CV for robust evaluation
  • Monitor Both Sets: Watch training AND validation performance

ベストプラクティス:

  • データが多いほど良い: 一般的にパフォーマンスが向上
  • 質 > 量: クリーンで正確なラベルが重要
  • 特徴エンジニアリング: 情報的な特徴を作成
  • シンプルから始める: まず単純なモデルを試す（ベースライン）
  • 交差検証: 堅牢な評価のためにk分割CVを使用
  • 両方のセットを監視: 訓練と検証のパフォーマンスを両方観察

Summary / まとめ

Key Takeaways:

  • Supervised learning uses labeled data to make predictions
  • Classification predicts categories; Regression predicts numbers
  • Always split data: Training (70-80%), Validation (10-15%), Test (10-15%)
  • Features (X) are inputs; Labels (y) are outputs
  • Avoid overfitting (too complex) and underfitting (too simple)
  • Real applications: Fraud detection, home prices, YouTube recommendations

重要ポイント:

  • 教師あり学習はラベル付きデータを使って予測を行う
  • 分類はカテゴリを予測；回帰は数値を予測
  • 常にデータを分割: 訓練（70-80%）、検証（10-15%）、テスト（10-15%）
  • 特徴（X）は入力；ラベル（y）は出力
  • 過学習（複雑すぎ）と学習不足（単純すぎ）を避ける
  • 実際の応用: 不正検出、住宅価格、YouTube推薦
