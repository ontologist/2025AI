What is Clustering? / クラスタリングとは？

Definition / 定義:

Clustering = Grouping similar items together without predefined labels

クラスタリング = 事前定義されたラベルなしで類似アイテムをグループ化すること

Key Concepts:

  • Cluster: A group of similar data points
  • Similarity: How close data points are to each other
  • Distance: Metric to measure dissimilarity
  • Centroid: The center point of a cluster

主要概念:

  • クラスタ: 類似したデータポイントのグループ
  • 類似性: データポイント同士がどれだけ近いか
  • 距離: 非類似性を測定する指標
  • 重心: クラスタの中心点

K-means Clustering Algorithm

K-meansクラスタリングアルゴリズム

How K-means Works / K-meansの仕組み:

  1. Choose K → Decide how many clusters you want
    Kを選択 → 必要なクラスタ数を決定
  2. Initialize Centroids → Randomly place K center points
    重心を初期化 → K個の中心点をランダムに配置
  3. Assign Points → Each data point joins nearest centroid's cluster
    ポイントを割り当て → 各データポイントは最も近い重心のクラスタに参加
  4. Update Centroids → Move centroids to cluster centers
    重心を更新 → 重心をクラスタの中心に移動
  5. Repeat → Steps 3-4 until centroids stop moving
    繰り返し → 重心が動かなくなるまでステップ3-4を繰り返す

Goal: Minimize distance between points and their cluster centroids

目標: ポイントとそのクラスタ重心間の距離を最小化

Choosing the Right K / 適切なKの選択

How Many Clusters Should You Create?

いくつのクラスタを作成すべきか？

Elbow Method

エルボー法

  • Plot error vs number of clusters
  • エラー対クラスタ数をプロット
  • Look for "elbow" point where improvement slows
  • 改善が遅くなる「肘」のポイントを探す

Domain Knowledge

ドメイン知識

  • Use business understanding
  • ビジネス理解を使用
  • Consider practical constraints
  • 実用的な制約を考慮

Example: Customer segmentation might use K=3 (high/medium/low value) or K=5 for more granularity

例: 顧客セグメンテーションはK=3（高/中/低価値）またはより細かくK=5を使用するかもしれません

Limitations of K-means Clustering

K-meansクラスタリングの限界

Challenges to Consider:

  • Must Choose K: Number of clusters must be specified beforehand
  • Sensitive to Outliers: Extreme values can distort clusters
  • Assumes Spherical Clusters: Works best for round, evenly-sized clusters
  • Random Initialization: Different starting points = different results
  • Scale Matters: Features with larger ranges dominate

考慮すべき課題:

  • Kの選択が必須: クラスタ数を事前に指定する必要がある
  • 外れ値に敏感: 極端な値がクラスタを歪める可能性
  • 球状クラスタを仮定: 丸く均等なサイズのクラスタに最適
  • ランダム初期化: 異なる開始点 = 異なる結果
  • スケールが重要: より大きな範囲の特徴が支配的

Solution: Normalize data, run multiple times, try other algorithms (DBSCAN, hierarchical)

解決策: データを正規化、複数回実行、他のアルゴリズムを試す（DBSCAN、階層的）
