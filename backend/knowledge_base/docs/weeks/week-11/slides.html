Decision Trees / 決定木

What is a Decision Tree?

A tree-like model that makes decisions by asking a series of questions

一連の質問をして意思決定を行うツリー状のモデル

Example: Should I play tennis today? Is it Sunny? ├─ Yes → Is Humidity High? │ ├─ Yes → Don't Play │ └─ No → Play └─ No → Is it Rainy? ├─ Yes → Is Wind Strong? │ ├─ Yes → Don't Play │ └─ No → Play └─ No → Play

How Decision Trees Work

決定木の仕組み

Building a Decision Tree:

  1. Start at Root: All training data
  2. Choose Best Feature: Split that best separates classes
  3. Create Branches: One for each feature value
  4. Repeat: Recursively split each branch
  5. Stop: When all examples in a node are same class

Splitting Criteria:

  • Gini Impurity
  • Information Gain (Entropy)

決定木の構築:

  1. ルートから開始: すべての訓練データ
  2. 最良の特徴を選択: クラスを最もよく分離する分割
  3. ブランチを作成: 各特徴値に対して1つ
  4. 繰り返し: 各ブランチを再帰的に分割
  5. 停止: ノード内のすべての例が同じクラスの時

分割基準:

  • ジニ不純度
  • 情報利得（エントロピー）

Real-World: Medical Diagnosis

実世界の例: 医療診断

Application: Disease diagnosis systems used in hospitals

Example: IBM Watson Health - ibm.com/watson-health

Decision Tree for Diagnosis:

  • Features: Symptoms, age, medical history, test results
  • Decision Path: Series of yes/no questions
  • Output: Disease classification or treatment recommendation
  • Advantage: Doctors can understand the reasoning
  • Interpretability: Can explain why a diagnosis was made

診断のための決定木:

  • 特徴: 症状、年齢、病歴、検査結果
  • 決定パス: 一連のはい/いいえの質問

  • 利点: 医師が推論を理解できる
  • 解釈可能性: なぜ診断が下されたか説明できる

Logistic Regression / ロジスティック回帰

What is Logistic Regression?

Despite its name, it's a CLASSIFICATION algorithm, not regression!

名前とは裏腹に、これは回帰ではなく分類アルゴリズムです！

How it Works:

  • Calculates probability of belonging to a class (0 to 1)
  • Uses sigmoid function: S(x) = 1 / (1 + e^-x)
  • If probability > 0.5 → Class 1, else → Class 0
  • Finds linear decision boundary

Best For:

  • Binary classification (2 classes)
  • When classes are linearly separable
  • Need probability estimates

仕組み:

  • クラスに属する確率を計算（0から1）
  • シグモイド関数を使用: S(x) = 1 / (1 + e^-x)
  • 確率 > 0.5 → クラス1、そうでなければ → クラス0
  • 線形決定境界を見つける

最適な用途:

  • 二値分類（2クラス）
  • クラスが線形分離可能な場合
  • 確率推定が必要な場合

Real-World: Credit Scoring

実世界の例: 信用スコアリング

Application: FICO Score, Credit Karma

Link: https://www.fico.com

Logistic Regression for Credit:

  • Goal: Predict if applicant will default on loan
  • Features: Income, debt, credit history, employment
  • Output: Probability of default (0-1)
  • Decision: Approve if P(default) < threshold
  • Advantage: Provides risk score, not just yes/no

Impact: Used by banks worldwide for loan decisions

信用のためのロジスティック回帰:

  • 目標: 申請者がローンでデフォルトするか予測
  • 特徴: 収入、負債、信用履歴、雇用
  • 出力: デフォルトの確率（0-1）
  • 決定: P(デフォルト) < 閾値なら承認
  • 利点: はい/いいえだけでなくリスクスコアを提供

影響: 世界中の銀行がローン決定に使用

Support Vector Machines (SVM)

サポートベクターマシン（SVM）

Core Idea / 中核的なアイデア:

Find the best boundary (hyperplane) that separates classes with maximum margin

最大マージンでクラスを分離する最良の境界（超平面）を見つける

Key Concepts:

  • Hyperplane: Decision boundary
  • Support Vectors: Data points closest to boundary
  • Margin: Distance between boundary and support vectors
  • Kernel Trick: Handle non-linear data

Strengths:

  • Effective in high dimensions
  • Works well with clear margins
  • Robust to overfitting

主要概念:

  • 超平面: 決定境界
  • サポートベクター: 境界に最も近いデータポイント
  • マージン: 境界とサポートベクター間の距離
  • カーネルトリック: 非線形データを処理

強み:

  • 高次元で効果的
  • 明確なマージンで良好に機能
  • 過学習に対してロバスト

Real-World: Image Classification

実世界の例: 画像分類

Application: Face detection, handwriting recognition

Example: Historical use in digit recognition (MNIST dataset)

SVM for Images:

  • Task: Classify images into categories
  • Features: Pixel values, image gradients, textures
  • Kernel: RBF (Radial Basis Function) for non-linear patterns
  • Application: Face detection in cameras, OCR systems
  • Success: Achieved high accuracy before deep learning era

画像のためのSVM:

  • タスク: 画像をカテゴリに分類
  • 特徴: ピクセル値、画像勾配、テクスチャ
  • カーネル: 非線形パターンのためのRBF（放射基底関数）
  • 応用: カメラの顔検出、OCRシステム
  • 成功: 深層学習時代以前に高精度を達成

Algorithm Comparison

アルゴリズム比較

Decision Trees

Pros / 利点:

  • Easy to understand and visualize
  • 理解と可視化が容易
  • Handles non-linear data
  • 非線形データを処理
  • No feature scaling needed
  • 特徴スケーリング不要

Cons / 欠点:

  • Prone to overfitting
  • 過学習しやすい
  • Unstable (small data changes = big tree changes)

Logistic Regression

Pros / 利点:

  • Provides probabilities
  • 確率を提供
  • Fast training
  • 高速訓練
  • Less prone to overfitting
  • 過学習しにくい

Cons / 欠点:

  • Assumes linear relationship
  • 線形関係を仮定
  • Cannot handle complex patterns

Real-World: Spam Detection (Revisited)

実世界の例: スパム検出（再訪）

Multiple Algorithms Combined

Link: Gmail Spam Filter

Gmail's Evolution:

  • Early Days: Logistic Regression (simple, fast)
  • Mid 2000s: SVM (better accuracy)
  • 2010s: Decision Trees + Ensemble Methods
  • Current: Deep Neural Networks

Why Multiple Algorithms?

  • Different algorithms catch different spam patterns
  • Ensemble methods combine strengths

Gmailの進化:

  • 初期: ロジスティック回帰（シンプル、高速）
  • 2000年代中頃: SVM（より高い精度）
  • 2010年代: 決定木 + アンサンブル法
  • 現在: 深層ニューラルネットワーク

なぜ複数のアルゴリズム？

  • 異なるアルゴリズムが異なるスパムパターンを捕捉
  • アンサンブル法が強みを組み合わせる

Choosing the Right Algorithm

適切なアルゴリズムの選択

Decision Guide:

  • Use Decision Trees when:
    • You need interpretability
    • Data has categorical features
    • Non-linear relationships
  • Use Logistic Regression when:
    • Need probability estimates
    • Linear decision boundary works
    • Want fast, simple model
  • Use SVM when:
    • High-dimensional data
    • Clear margin of separation
    • Need robust performance

決定ガイド:

  • 決定木を使用する場合:
    • 解釈可能性が必要
    • データにカテゴリ特徴がある
    • 非線形関係
  • ロジスティック回帰を使用する場合:
    • 確率推定が必要
    • 線形決定境界で機能する
    • 高速でシンプルなモデルが必要
  • SVMを使用する場合:
    • 高次元データ
    • 明確な分離マージン
    • ロバストなパフォーマンスが必要
