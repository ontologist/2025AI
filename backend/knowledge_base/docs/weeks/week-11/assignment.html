第11週課題

Comparing Classification Algorithms

分類アルゴリズムの比較

Due Date: Before next class (Week 12)

Estimated Time: 50-60 minutes

Grade Weight: Part of 40% Individual Reports

締切: 次回授業前（第12週）

Learning Objectives / 学習目標

  • Understand how different classification algorithms work
  • Compare Decision Trees, Logistic Regression, and SVM
  • Choose appropriate algorithms for different problems
  • Analyze real-world classification applications
  • 異なる分類アルゴリズムがどのように機能するかを理解する
  • 決定木、ロジスティック回帰、SVMを比較する
  • 異なる問題に適切なアルゴリズムを選択する
  • 実世界の分類応用を分析する

Part 1: Algorithm Comparison Chart (35 points)

パート1: アルゴリズム比較チャート（35点）

Question 1.1 (35 points)

Create a comprehensive comparison table for the three classification algorithms we studied: Decision Trees, Logistic Regression, and Support Vector Machines (SVM).

Your table should compare them across these dimensions:

  1. How it works: Brief explanation of the core mechanism (2-3 sentences each)
  2. Decision boundary: What type of boundary does it create?
  3. Interpretability: How easy is it to understand why a decision was made? (Scale: High/Medium/Low + explanation)
  4. Strengths: What is each algorithm particularly good at? (3-4 points)
  5. Weaknesses: What are the limitations? (3-4 points)
  6. Best use cases: When should you use this algorithm? (2-3 scenarios)
  7. NOT good for: When should you avoid this algorithm? (2-3 scenarios)
  8. Real-world example: One specific application that uses this algorithm

Present as a well-organized table. You may use landscape orientation if needed.

質問1.1（35点）

私たちが学んだ3つの分類アルゴリズムの包括的な比較表を作成してください: 決定木、ロジスティック回帰、サポートベクターマシン（SVM）。

表は以下の次元でそれらを比較する必要があります:

  1. 仕組み: コアメカニズムの簡潔な説明（各2-3文）
  2. 決定境界: どのタイプの境界を作成するか？
  3. 解釈可能性: なぜ決定が下されたかを理解するのはどれくらい簡単か？（スケール: 高/中/低 + 説明）
  4. 強み: 各アルゴリズムは特に何が得意か？（3-4点）
  5. 弱点: 限界は何か？（3-4点）
  6. 最良の使用例: このアルゴリズムをいつ使用すべきか？（2-3のシナリオ）
  7. 適していない: このアルゴリズムをいつ避けるべきか？（2-3のシナリオ）
  8. 実世界の例: このアルゴリズムを使用する1つの具体的な応用

よく整理された表として提示してください。必要に応じて横向きを使用できます。

Part 2: Algorithm Selection for Problems (35 points)

パート2: 問題のためのアルゴリズム選択（35点）

Problem Scenarios

You are an ML consultant. For each of the following problems, recommend which classification algorithm(s) to use and justify your choice.

問題シナリオ

あなたはMLコンサルタントです。以下の各問題について、どの分類アルゴリズムを使用するかを推奨し、選択を正当化してください。

Question 2.1 (15 points)

Scenario A: Medical Diagnosis System

A hospital wants to build a system that helps doctors diagnose a disease based on patient symptoms and test results. The system must:

  • Provide explanations for its decisions (doctors need to understand WHY)
  • Handle both categorical (yes/no symptoms) and numerical (test values) features
  • Be interpretable for regulatory approval
  • Predict whether patient has disease A, B, or C, or is healthy (4 classes)

Answer:

  1. Which algorithm would you recommend? (You may suggest multiple)
  2. Why is this algorithm the best choice for this specific problem?
  3. What are the potential risks if you chose the wrong algorithm?
  4. How would you validate that your model is working correctly before deployment?

Write 200-250 words with clear reasoning.

質問2.1（15点）

シナリオA: 医療診断システム

病院は、患者の症状と検査結果に基づいて医師が疾患を診断するのを助けるシステムを構築したいと考えています。システムは以下を満たす必要があります:

  • その決定の説明を提供する（医師はなぜかを理解する必要がある）
  • カテゴリ的（はい/いいえの症状）と数値的（検査値）の両方の特徴を処理する
  • 規制承認のために解釈可能である
  • 患者が疾患A、B、またはCを持っているか、健康であるかを予測する（4クラス）

回答:

  1. どのアルゴリズムを推奨しますか？（複数を提案してもよい）
  2. なぜこのアルゴリズムがこの特定の問題に最良の選択ですか？
  3. 間違ったアルゴリズムを選択した場合の潜在的なリスクは何ですか？
  4. 展開前にモデルが正しく機能していることをどのように検証しますか？

明確な推論で200-250語で記述してください。

Question 2.2 (20 points)

Scenario B: Large-Scale Image Classification

A tech company wants to classify images into 1000 different categories (cats, dogs, cars, buildings, etc.). They have:

  • 10 million labeled images
  • High-dimensional feature vectors (thousands of pixel values)
  • Need for high accuracy
  • Computational resources available
  • Less concern about interpretability

Answer:

  1. Would you use Decision Trees, Logistic Regression, or SVM? Explain why or why not for EACH algorithm.
  2. What modifications or variations of these algorithms might work better?
  3. What other type of algorithm might be even better for this problem? (Hint: Think beyond this week's content)
  4. How do the data characteristics (high-dimensional, large dataset) influence your choice?

Write 250-300 words analyzing each algorithm's suitability.

質問2.2（20点）

シナリオB: 大規模画像分類

テクノロジー企業は、画像を1000の異なるカテゴリ（猫、犬、車、建物など）に分類したいと考えています。彼らは以下を持っています:

  • 1000万のラベル付き画像
  • 高次元特徴ベクトル（数千のピクセル値）
  • 高精度の必要性
  • 利用可能な計算リソース
  • 解釈可能性への懸念は少ない

回答:

  1. 決定木、ロジスティック回帰、またはSVMを使用しますか？各アルゴリズムについてなぜか、なぜそうでないかを説明してください。
  2. これらのアルゴリズムのどのような修正や変種がより良く機能する可能性がありますか？
  3. この問題にさらに優れた他のタイプのアルゴリズムは何ですか？（ヒント: 今週の内容を超えて考える）
  4. データ特性（高次元、大規模データセット）はあなたの選択にどのように影響しますか？

各アルゴリズムの適合性を分析して250-300語で記述してください。

Part 3: Decision Tree Walkthrough (30 points)

パート3: 決定木のウォークスルー（30点）

Scenario: University Admission Decision

Create a decision tree for predicting whether a student will be admitted to a university program.

シナリオ: 大学入学決定

学生が大学プログラムに入学するかどうかを予測するための決定木を作成してください。

Question 3.1 (30 points)

Design a decision tree (3-4 levels deep) that could be used for university admission. Your tree should:

  1. Identify Features (5 points): List 4-5 features that would be relevant for admission decisions (e.g., GPA, test scores, extracurricular activities, etc.)
  2. Draw the Tree (15 points): Create a decision tree diagram showing:
    • Root node: Which feature would you split on first? Why?
    • Internal nodes: Subsequent decision points
    • Leaf nodes: Final decisions (Admit/Reject/Waitlist)
    • Decision rules at each split (e.g., "GPA > 3.5?")
    You can hand-draw and scan/photo, or create digitally.
  3. Explain Your Logic (10 points): Write 150-200 words explaining:
    • Why did you choose that feature for the root node?
    • How did you decide the order of features?
    • What makes a "good" split in a decision tree?
    • What are potential problems with this tree design?

質問3.1（30点）

大学入学に使用できる決定木（3-4レベルの深さ）を設計してください。あなたの木は以下を満たす必要があります:

  1. 特徴を特定する（5点）: 入学決定に関連する4-5の特徴をリストする（例: GPA、テストスコア、課外活動など）
  2. 木を描く（15点）: 以下を示す決定木図を作成する:
    • ルートノード: どの特徴で最初に分割しますか？なぜですか？
    • 内部ノード: その後の決定ポイント
    • リーフノード: 最終決定（入学/不合格/待機リスト）
    • 各分割での決定ルール（例: 「GPA > 3.5?」）
    手描きしてスキャン/撮影するか、デジタルで作成できます。
  3. ロジックを説明する（10点）: 150-200語で以下を説明する:
    • なぜルートノードにその特徴を選択したのですか？
    • 特徴の順序をどのように決定しましたか？
    • 決定木における「良い」分割とは何ですか？
    • この木の設計における潜在的な問題は何ですか？
